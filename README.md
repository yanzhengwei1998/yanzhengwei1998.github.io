# 考公考编资料库

这是一个用于事业单位、公务员、公基、申论和面试备考的 Markdown 知识整理仓库。

仓库目标是把零散资料整理成结构清晰、方便复习、便于持续追加的分类文档。所有资料默认只做知识整理，不额外添加资料出处类、压缩提纲类、未来补充类章节。

## 目录结构

| 目录 | 内容 |
| --- | --- |
| `政治/` | 政治理论、党史、党建、党的会议等内容 |
| `时政热点/` | 年度时政、重要会议、规划政策、热点专题 |
| `法律/` | 法律基础、宪法、民法、行政法、刑法、诉讼法、新法速递 |
| `省情市情/` | 吉林省省情、市情、州情及相关政策 |

## 重点文档

| 文档 | 用途 |
| --- | --- |
| `AGENTS.md` | 项目整理规则，跨电脑和跨 Codex 会话时优先阅读 |
| `时政热点/2026年时政.md` | 2026 年全年时政滚动整理文档 |
| `政治/党史信息.md` | 党史时间线、重要会议、重要事件、精神谱系 |
| `省情市情/吉林省省情市情.md` | 吉林省及各市州省情市情整理 |
| `法律/README.md` | 法律目录索引 |

## 整理规则

资料整理遵循以下原则：

1. 使用 Markdown 编写。
2. 按类别放入对应目录。
3. 重要时间、地点、数字、文件名称等使用重点标识。
4. 内容优先整理为表格、关键词、考点速记、易混点、答题角度。
5. 不主动添加资料出处类、压缩提纲类、未来补充类章节。
6. 新增内容优先追加到已有对应文档，必要时再新建专题文档。

## Git 工作流

每次处理资料前：

```powershell
git pull
```

然后阅读：

```text
AGENTS.md
```

整理完成后：

```powershell
git status
git add .
git commit -m "清晰的提交说明"
git push
```

Windows 如果 `git` 不在 PATH 中，可以使用：

```powershell
& 'C:\Program Files\Git\cmd\git.exe' status
```

## 跨电脑使用

在新电脑上克隆仓库：

```powershell
git clone https://github.com/yanzhengwei1998/KaoGongKaoBian.git
cd KaoGongKaoBian
git pull
```

然后用 Codex 打开仓库目录，并要求先阅读 `AGENTS.md`，再继续整理资料。
