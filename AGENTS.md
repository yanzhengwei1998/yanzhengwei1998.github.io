# 考公考编资料整理规则

本文件用于让不同电脑、不同 Codex 会话都能延续同一套资料整理方式。

## 一、固定工作流程

1. 如果用户在对话中直接输入知识点，先核验知识点正确性，再写入对应 Markdown 文档。
2. 根据资料类别整理到对应 Markdown 文档。
3. 不随意删除已有资料；新增内容优先追加、归并或按既有结构重排。
4. 所有目录下的所有资料文档默认只进行知识整理，不添加资料出处类、压缩提纲类、未来补充类章节；除非用户明确要求。
5. Git 相关操作不再默认执行；只有用户明确指定时才进行。

## 二、资料分类规则

- 政治理论、党史、党建、党的会议资料：放入 `政治/`
- 法律基础、宪法、民法、行政法、刑法、诉讼法、最新法律法规：放入 `法律/`
- 时政热点、年度时政：放入 `时政热点/`
- 吉林省省情、市情、州情：放入 `省情市情/`
- 岗位、考试公告、报名信息：后续可新建 `岗位信息/`
- 考试大纲、备考资料：后续可新建 `考试资料/`
- 错题、薄弱点、复盘：后续可新建 `错题复盘/`

## 三、已有重要文档规则

### 1. `时政热点/2026年时政.md`

- 作为 2026 年全年时政滚动文档。
- 后续新增的 2026 年时政继续追加到这个文件。
- 按月份或专题整理，不再另建 `2026年时政1-4月.md` 这类文件。
- 动态信息要标注时间，重要时间、数字、文件名称要加重点标识。

### 2. `省情市情/吉林省省情市情.md`

- 按市州整理知识。
- 每个市州统一包含：
  - 基本市情
  - 产业和文旅
  - 近一年政策
  - 考点速记
  - 面试关键词
- 后续新增吉林省或各市州政策时，优先追加到对应市州小节。

### 3. `政治/党史信息.md`

- 时间、地点要重点标识。
- 统一使用格式：
  - `【重点时间】**xxxx年xx月**`
  - `【重点地点】**地点**`
- 内容优先整理为时间线、重要会议、重要事件、精神谱系、高频考点和易混点。

### 4. `时政热点/党的二十大.md`

- 保持“报告结构、核心理论、高频考点、申论面试角度、易混点”的结构。
- 后续补充二十大相关内容时，归入对应章节。

### 5. `时政热点/党的二十届历次全会.md`

- 按一中、二中、三中、四中等历次全会整理。
- 新增全会内容时，按时间顺序追加，并补充总览表和易混点。

### 6. `时政热点/十五五规划.md`

- 区分《建议》和《纲要》。
- 后续补充“十五五”政策时，放入对应领域章节，避免和年度时政重复堆叠。

### 7. `法律/`

- 法律相关内容按不同部门法或专题拆分为不同 Markdown 文档。
- 可优先建立：
  - `README.md`
  - `法律基础.md`
  - `宪法.md`
  - `民法.md`
  - `行政法.md`
  - `刑法.md`
  - `诉讼法.md`
  - `新法速递.md`
- 法律内容属于动态且严谨信息，整理时可优先参考国家法律法规数据库、中国人大网、司法部等官方渠道进行内部核验。
- 法律条文不要大段照抄，优先整理为概念、原则、适用条件、易混点、考试速记和案例化理解。
- 涉及施行时间、修订时间、机关权限、法律责任等内容，要使用重点标识。
- `法律/README.md` 作为法律目录索引；具体知识点写入对应专题文档，不把所有内容堆在一个总文档里。

## 四、整理风格

1. 使用 Markdown。
2. 内容要适合事业单位、公务员、公基、申论、面试复习。
3. 不简单复制原文，要提炼为：
   - 基本信息
   - 高频考点
   - 易混点
   - 申论/面试可用角度
4. 动态政策、时政信息要尽量标注具体时间。
5. 对时间、地点、数字、文件名称、会议名称、法律法规施行时间等高频考点加重点标识。
6. 默认只整理知识点本身，不主动添加资料出处类、压缩提纲类、未来补充类章节；除非用户明确要求。
7. 可优先使用官方渠道进行核验；用户提供的 PDF、网页、材料，可在必要时简要注明材料名称，但不单独设置资料出处章节。
8. 避免无意义堆砌，尽量整理成表格、速记、关键词和答题框架。

## 五、重点标识规则

常用格式：

- 时间：`【重点时间】**2026年1月1日**`
- 地点：`【重点地点】**北京**`
- 数字：`【重点】**8个地级市、1个自治州**`
- 文件：`【重点文件】**《中共中央关于进一步全面深化改革、推进中国式现代化的决定》**`
- 政策：`【重点政策】**长春现代化都市圈建设**`

## 六、核验规则

1. 用户提供文件时，可在文档开头简要注明材料名称。
2. 用户在对话中直接输入知识点时，必须先核验正确性，再整理写入对应文档。
3. 涉及时政、政策、法律、领导活动、最新数据等动态信息，应尽量使用官方渠道核验。
4. 如果用户输入的知识点存在错误，应先修正为准确表述；如果暂时无法确认，应先标注“待确认”，不要当作确定知识点写入。
5. 如果某些内容来自备考资料而未实时核验，可在文档说明中简要提示“依据用户提供资料整理”。
6. 国际要闻、数据类内容变化快，整理时要标注时间；不默认添加资料出处章节。
